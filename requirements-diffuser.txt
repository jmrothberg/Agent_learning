# GPU extras layered AFTER scripts/install_diffuser.sh installs torch + diffusers.
# That script runs `pip install -r requirements-diffuser.txt` as its final step
# (soundfile, torchsde for Stable Audio; imageio for Linux Wan MP4 export;
# mflux on macOS for FLUX2-klein sprites).
#
# Optional GPU stack for sprite generation + Stable Audio Open (sounds.py)
# + Linux Wan cutscene MP4 export (scripts/generate_video.py).
#   macOS (Apple Silicon): FLUX2-klein-9B-mlx-8bit via mflux CLI ONLY (never Z-Image).
#   Linux: Z-Image-Turbo (diffusers) + SD-Turbo img2img chain.
#
# IMPORTANT: this file does NOT pin torch / diffusers because the
# correct flavor depends on your CUDA version. The install script
# pulls torch nightly (cu130 — needed for current GPUs) and diffusers
# from git HEAD (Z-Image-Turbo support landed too recently for any
# tagged release). The lines below are the *plain* deps that come
# from PyPI normally.
#
# Manual-only pip path if torch/diffusers are already installed:
#   .venv/bin/pip install -r requirements-diffuser.txt
#
# After install, the FIRST <assets> request in a session will:
#   macOS: use FLUX2-klein (mflux) when FLUX2-klein-9B-mlx-8bit is under
#     DIFFUSION_MODELS_DIR (setup.sh / .env) — ~10-15s first sprite, then faster.
#     If weights or mflux are missing, sprite gen is skipped (no Z-Image download).
#   Linux: Z-Image-Turbo under DIFFUSION_MODELS_DIR or HF cache (~32 GB
#     download if missing), load into GPU VRAM (~30-60s first time), ~2-4s/sprite.
#
# All in-process. No HTTP server, no subprocess.

transformers>=4.40.0
accelerate>=0.30.0
safetensors>=0.4.0
# soundfile (libsndfile) — used by sounds.py to encode Stable Audio
# Open output as OGG-Vorbis. Cross-platform wheel on PyPI; no system
# deps on macOS or Linux. ~0.5 MB.
soundfile>=0.12.0
# torchsde — required by Stable Audio Open's default scheduler
# (CosineDPMSolverMultistepScheduler). Without it, pipeline load fails
# with `ImportError: CosineDPMSolverMultistepScheduler requires the
# torchsde library`. The library has a known recursion-depth issue on
# the last timestep that sounds.py works around by raising the Python
# recursion limit before each generate() call (see _lazy_init).
torchsde>=0.2.5
# imageio + ffmpeg plugin — Linux Wan video (scripts/generate_video.py) uses
# diffusers.export_to_video, which needs imageio[ffmpeg] (or opencv). Without
# these, inference can succeed and then crash on MP4 write. macOS mlx-gen
# writes MP4 itself; the wheels are harmless there and keep one shared file.
imageio>=2.34.0
imageio-ffmpeg>=0.5.0
# torch and diffusers — install via scripts/install_diffuser.sh
# (cu130 nightly + diffusers git HEAD).
#
# mflux — FLUX2-klein sprite CLI on macOS (assets.py Flux2KleinMfluxGenerator).
# install_diffuser.sh pip-installs this on Darwin only; Linux/CUDA hosts keep
# Z-Image-Turbo + SD-Turbo img2img. Put FLUX2-klein-9B-mlx-8bit under
# DIFFUSION_MODELS_DIR (setup.sh writes .env when it finds local weights).
mflux>=0.18.0; sys_platform == 'darwin'
